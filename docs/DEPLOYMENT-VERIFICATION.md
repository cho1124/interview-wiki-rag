## 교훈

1. 적대적 검증은 설계/코드 수준에서 강력하지만, **배포 환경 결함은 별도 계층**
2. 프레임워크 메이저 버전 변경(Tailwind v4, AI SDK v6)은 검증 시점에 존재하지 않던 변화
3. 7건 중 **2건은 적대적 검증에서 잡을 수 있었음** → 검증 커버리지 개선 여지
